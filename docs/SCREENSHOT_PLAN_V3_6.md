# Screenshot Plan v3.6

Last updated: 2026-06-08

Use this plan to capture Google Play screenshots after the v3.6 release candidate is installed with clean demo data. Do not generate images from this document; use it as a capture and creative brief.

## Capture Rules

- Use a release-like build without debug overlays.
- Avoid private data, test credentials, broken AI states, unavailable billing states, and guaranteed-success claims.
- Use short, store-friendly captions.
- Show active Premium packs as available: SQL Interview Prep, QA Advanced, Automation Testing Basics.
- Show Python, JavaScript, Business English, and Technical Interview Prep only as coming soon unless full content is added later.
- Prefer clean demo progress with at least two weak topics.

- [ ] Selected path is either IT / QA Interview Prep or SQL Interview Prep.

## Phone Notes

- Capture portrait screenshots.
- Check text wrapping and bottom navigation visibility.
- Avoid scrolling screenshots unless Play Console image dimensions are prepared externally.
- First three screenshots should communicate app value without needing users to zoom.

## Tablet Notes

- Tablet screenshots are optional.
- Use Home, Progress, Premium Study Packs, or Premium screen if the layout is readable.
- Avoid dense quiz/flashcard text if the tablet crop makes it feel sparse.
